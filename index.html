Copilot Tuning Reference

Why Copilot Tuning Beats a Custom Communications Agent

FactorCustom AgentCopilot Tuning
Email creationSeparate tool – copy/paste into OutlookNative in Outlook compose window
MaintenanceYou maintain itMicrosoft maintains it
IntegrationLimitedFull M365 (emails, files, calendar, contacts)
ScalabilityDeploy/manage per userAssign a licence, done
CostDev + maintenance + API feesAlready in Copilot subscription
UpdatesYou build featuresMicrosoft ships automatically
Tone controlHardcoded rules, less flexibleFlexible per context, stays on-brand

What IAS Can Do Now (Before Tuning Is Available)

Quick Win 1: Outlook Draft Instructions

Every user sets persistent drafting preferences in Copilot settings. Example:

“Write emails in a warm, professional tone. Use plain language – avoid insurance jargon. Keep paragraphs to 2–3 sentences. Use bullet points for complex information. Always include a greeting using the recipient’s first name. Close with ‘Kind regards’ followed by the IAS signature block.”

Quick Win 2: Prompt Template Library

Shared set of ready-to-use prompts for common IAS communications: claims acknowledgment, policy renewal reminders, internal updates, client onboarding.

Quick Win 3: Organisation-Wide Default Tone

Admin capability to set a tenant-wide default tone for Copilot. All AI-drafted content matches IAS’s voice by default.

What IAS Needs to Prepare for Copilot Tuning

  1. 20+ writing samples – best client emails, well-crafted internal comms that capture IAS’s ideal voice
  2. Style guide covering tone, vocabulary, formatting, compliance requirements
  3. A champion (Leah) to manage the tuning process and test outputs

Bottom line: IAS is already paying for Copilot. Building a separate agent means paying twice for the same outcome, with inferior integration and ongoing maintenance. The smart move is to make Copilot work in IAS’s voice.
